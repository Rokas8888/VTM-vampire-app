Scene Maker — Full Improvement Plan

All 7 improvements: design revert + 6 UX wins + 3D visual quality. Here's the full approach before we write the spec.

Part 1 — Design Revert
🎨
Remove Gothic Overhaul → VTM App Design Fast
Strip out the injected GOTHIC_CSS block, all Cormorant/UnifrakturCook fonts, gold CSS variables, corner SVG ornaments, and fraktur branding. Replace with standard Tailwind classes matching the rest of the app — bg-void, border-void-border, text-blood, Cinzel for headers, Inter for body.
No new behaviour — pure visual cleanup. The 3D canvas itself stays the same.
Part 2 — Visual Quality (3D)
✨
Post-Processing: SSAO + Bloom Medium
SSAO (Ambient Occlusion) — the single biggest quality jump. Right now objects look like they're floating because there's no contact shadow between them. SSAO adds subtle darkening where surfaces meet walls, floors, and furniture. Makes assets look "grounded."

Bloom — selective glow only on lit objects (torches, candles). Torches get a warm orange halo; the rest of the scene stays sharp. Uses @react-three/postprocessing which wraps the canvas.
Install @react-three/postprocessing. Wrap canvas in <EffectComposer> with <SSAO radius=0.5 intensity=40> + <Bloom luminanceThreshold=1 mipmapBlur>. Torch/candle materials get emissiveIntensity > 1 to catch bloom.
💡
Better Lighting & Shadows Fast
Current hemisphere light is flat. Add a warmer key light + cooler fill light to create depth. Increase shadow map resolution from 1024 to 2048. Add subtle ground plane reflection using a dark MeshStandardMaterial with low roughness.
Tweak AtmosphereLights component — raise shadow-mapSize to 2048, adjust hemisphere sky/ground colours, add a second fill pointLight. No new dependencies.
Part 3 — UX Improvements
🔍
A — Asset Search / Filter Fast
203 models in a scrollable list is painful. Add a text input at the top of the left panel. Typing filters all category items by name in real time. Matching categories expand automatically; empty categories hide.
One searchTerm state variable. Filter Object.entries(ASSET_CATEGORIES) on render. No backend needed.
↩️
B — Undo / Redo (Ctrl+Z / Ctrl+Y) Medium
Every paint or erase action saves the previous grid state to a history stack. Ctrl+Z pops the last state; Ctrl+Y redoes. Max 40 snapshots to keep memory reasonable (28×28 = 784 cells per snapshot).
historyRef = array of grid snapshots. On every cell change push current grid before mutating. Keyboard listener for Ctrl+Z/Y. No performance issue — each snapshot is a plain JS array of small objects.
🔄
C — Object Rotation (R key) Medium
Press R to cycle the active brush through 0° / 90° / 180° / 270°. The rotation is stored per placed tile, so each cell remembers its own rotation. Essential for aligning doors, furniture against walls, etc.
Add brushRotation state (0, π/2, π, 3π/2). Cell data gains a rotY field per slot (floor/wall/object). Tile component reads cell.object.rotY. Keyboard listener for R key.
👻
D — Ghost Preview on Hover Medium
As you move the cursor over the grid, a translucent (50% opacity) version of the selected asset appears under the mouse. You always know exactly what and where you're about to place. Disappears when no asset is selected or in erase mode.
Track hoveredIdx in state. Render an extra <Tile> at that position with all materials set to transparent=true, opacity=0.45 via scene.traverse. Use a separate useMemo key so it doesn't interfere with real tiles.
🖌️
E — Better Fill Tools Medium
Fill Floor — already exists. Keep it.
Rectangle Fill — hold Shift + drag to mark a rectangle, release to fill every cell in that area with the selected asset.
Clear Section — hold Shift in Erase mode to drag-erase a rectangle area at once.
Add shiftDragStart ref. On Shift+pointerdown record start cell. On Shift+pointerup calculate bounding box and bulk-update grid. Visual feedback: highlight the selected rectangle during drag (coloured hit planes).
💾
F — Save / Load Scene Complex
Scenes are currently lost on page refresh. Add save/load to the database per GM group. The GM can name a scene, save it, and load it back any time. One scene per group (can expand to multiple later).
Backend: New scenes table — id, group_id (FK), name, data (JSONB), updated_at. Alembic migration 023. Two endpoints: GET /api/scenes/{group_id} + PUT /api/scenes/{group_id}.

Frontend: Save button in topbar. On load, fetch scene for current group. Auto-save every 30s if grid changed. Scene data = grid array serialised as JSON.
