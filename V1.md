# Vampire Scriptorium — Feature List

---

## Version 1.0 — Completed

### Foundation
- [x] User registration with role selection (Player / Game Master)
- [x] JWT authentication (access + refresh tokens)
- [x] Role-based routing — players, GMs, and admins land on different dashboards
- [x] Admin account creation via CLI
- [x] PostgreSQL database with Alembic migrations
- [x] Docker Compose setup (DB + backend + frontend in one command)
- [x] Cloudflare Tunnel support for sharing with players (documented in README)

### Game Data
- [x] Full V5 game data seeded: 16 clans, all disciplines, powers, predator types, merits, flaws, backgrounds
- [x] Info popups throughout the app (every selectable element has description + rules)
- [x] Admin seed button to restore game data from the panel

### Character Creation Wizard (10 steps)
- [x] Step 1 — Core concept: name, concept, ambition, desire
- [x] Step 2 — Clan selection with info popups
- [x] Step 3 — Predator type (auto-applies discipline dot, specialty, advantages/flaws)
- [x] Step 4 — Attribute distribution (enforces V5 rules: one at 4, one at 1, etc.)
- [x] Step 5 — Skill distribution (Jack of All Trades / Balanced / Specialist)
- [x] Step 6 — Discipline selection and power assignment
- [x] Step 7 — Advantages: backgrounds, merits, flaws
- [x] Step 8 — Beliefs: convictions and touchstones
- [x] Step 9 — Humanity (auto-modified by predator type)
- [x] Step 10 — Generation selection and full review
- [x] Save as draft at any step, resume later
- [x] Validation before proceeding at each step

### Player Dashboard
- [x] Character grid — all your vampires at a glance
- [x] Full character sheet view (attributes, skills, disciplines, powers, merits, flaws, backgrounds, convictions)
- [x] Health and willpower tracks (superficial / aggravated damage)
- [x] Hunger dots
- [x] Edit profile (biography, haven location, haven description, notes)
- [x] Session save (persists health/willpower/hunger changes between sessions)
- [x] Experience points — award XP, spend XP to improve traits
- [x] Improve stats mode — spend XP on attributes, skills, disciplines directly from the sheet
- [x] Weapons and possessions management
- [x] Delete character (with type-to-confirm protection)
- [x] V5 dice roller with hunger dice, messy critical and bestial failure detection
- [x] Per-die throw animation (Web Animations API)

### Player Directory
- [x] Browse all completed characters across the whole app
- [x] Filter by name, clan, generation
- [x] Click any character to view full read-only sheet

### GM Dashboard
- [x] Create and manage multiple groups
- [x] Search and add players to groups (two-step: pick user → pick character)
- [x] Remove players from groups
- [x] Adaptive character grid (layout adjusts for 1–7+ characters)
- [x] Character cards showing health, willpower, hunger, top skills
- [x] Drill-down to full read-only character sheet for any group member
- [x] Monster / NPC creator per group (name, type, health, attributes, attack, special abilities, notes)
- [x] Dice roller with manual hunger control
- [x] Mobile-responsive sidebar (collapsible on small screens)

### Admin Panel
- [x] System stats overview (users, groups, characters, monsters)
- [x] Full user registry with search
- [x] Change user roles (player ↔ gm ↔ admin)
- [x] Activate / deactivate accounts
- [x] Delete users (with confirmation)
- [x] Seed / restore game data button

### Design & UX
- [x] Dark gothic theme throughout (Cinzel font, deep reds, near-black backgrounds)
- [x] Gothic castle parallax login screen
- [x] Animated TV static noise overlay on login
- [x] Blood drip animations on login screen
- [x] Animated dice throw physics (per-die randomised trajectories)
- [x] Satanic ritual table for dice tray
- [x] Consistent loading states (gothic pulsing text)
- [x] Confirmation modals for all destructive actions
- [x] Responsive layout (tested on desktop and mobile)

## Version 1.1 — Planned Features

### Session Mode *(High priority)*
- [x] GM-only live session screen — full coterie status at a glance
- [x] Health, hunger, willpower for all characters in one view
- [x] Real-time reflection of player changes during play (polls every 5 s)
- [x] Designed for second monitor / tablet use during sessions

### Dice Roll History *(High priority)*
- [x] Persistent roll log — saves every roll to database
- [x] Player sees their own last 10 rolls in the dice modal history panel
- [x] GM dice roll history endpoint available (group history)

### Condition / Status Tracker *(Medium priority)*
- [x] GM applies conditions to characters (Impaired, Blinded, Frightened, Staked, etc.)
- [x] Players see active conditions on their sheet (shown in character sheet header)
- [x] Conditions clear manually by GM; preset list + custom name + severity (mild/moderate/severe)

### Chronicle Notes *(Medium priority)*
- [ ] Shared notepad per group
- [ ] GM writes session summaries
- [ ] Players can add personal notes
- [ ] Persists across sessions

### Blood Potency / Resonance Tracker *(Medium priority)*
- [ ] GM tracks NPCs fed on during session
- [ ] Records resonance type and potency level
- [ ] Quick-add panel during play

### NPC Rolodex *(Medium priority)*
- [ ] GM maintains a named NPC database per chronicle
- [ ] Fields: name, clan, status, relationship to coterie, notes
- [ ] Searchable — separate from combat monsters

### Character Portrait Upload *(Medium priority)*
- [ ] Players upload an image for their vampire
- [ ] Shown on character sheet header and GM grid cards
- [ ] File stored locally on the server

### Toast Notifications *(Low priority)*
- [x] Floating toast system — success / error / info variants
- [x] Auto-dismisses after 3.5 seconds, slide-in animation
- [x] Available app-wide via useToast() hook

### Export Character Sheet as PDF *(Low priority)*
- [ ] One-click export of a character sheet to a printable PDF
- [ ] Styled to match the V5 aesthetic

### Mobile PWA *(Low priority)*
- [ ] App installable on phone from the browser
- [ ] Works offline for character sheet viewing
- [ ] Manifest + service worker setup
